First the important info on the libs.

1.  pyvidplayer2
2.  pygame
3.  openCV with CUDA support compiled in.
4.  Other libs that are likely not directly responsible.

Overview of the setup.

There are a number of classes or relatively important files/functions:

1. FindVideos(opts) Class used to scan subdirectories for playable media, or for valid Playlists to load, or for  files that point to specific media to play.  Does a comprehensive test on valid playable media.  The function creates a master playlist called PlayList which is a list containing the path/filename of the media to play on each line.

2. PlayVideo:  Class which plays the video, creates an instance of the EventHandler class, initializes  and plays the video.  Also contains all the methods to interact with the video,  Most GUI controls (but not all).

3. EventHandler Class which is the pygame event handler which handles most pygame events.

4. cmdLineOptions()  (in file cmdLineOpts.py).  Responsible for  parsing (via argparse) command line arguments.


5. pyv2.py :
    A. Contains the function main().
    B. Responsible for getting all CLI options.  It is called as: opts = cmdLineOptions().  Opts contains all of our command line arguments.
    C. Creates an instance of the Class FindVideos.  It is called as : Videos = FindVideos(opts)
    D. Creates an instance of the class PlayVideo called playVid. It is called by: playVid = PlayVideo(opts, Videos.videoList, bcolors) where: opts contains all the command line arguments, Videos.videoList  is the master video playlist that  the FindVideo Class creates.
    E. Creates an instance of the pygame event handler from the EventHandler class: eventHandler = EventHandler(playVid)
    F. Finally, play all the videos that are specified in playVid.videoList.


    In the PlayVideo Class, there are two main methods of concern in its everyday operation:  playVideo(self, video) and  play(self, eventHandler).

    The method named playVideo has  a path/filename passed to it.  It is responsible for initializing the  video using pyvidplayer2 and this returns a video handler to the calling method (play).

    Structure of the PlayVideo.play(self, eventHandler) method.

It is easier to just  post the revelant lines in the  play method:

    def play(self, eventHandler):

        while True:
            self.currVidIndx = -1
            while self.currVidIndx < len(self.videoList):
                self.forwardsFlag = False
                self.win.fill((0, 0, 0))

                if not self.backwardsFlag:
                    if not self.opts.loop_flag:
                        self.currVidIndx += 1
                else:
                    if self.currVidIndx < 0:
                        self.currVidIndx = 0
                    self.backwardsFlag = False
                    if self.currVidIndx > 0:
                        self.currVidIndx -= 1
                if self.currVidIndx == len(self.videoList):
                    break

                # .gif animation requires that we do not display the video splash.
                if self.videoList[self.currVidIndx].lower().endswith(".gif"):
                    self.disableSplash = True
                else:
                    self.disableSplash = False

                self.reset_OSD_tracking()
                self.reset_video_info_splash()

                '''
                Play the video
                This is the call to the playVideo instance
                '''
                self.vid = self.playVideo(self.videoList[self.currVidIndx])
                if self.vid is None:
                    continue

                print(f"Playing {self.currVidIndx+1} of {len(self.videoList  )}: {self.vid.name}{self.vid.ext}")
                # Setup video splash.
                if not self.disableSplash:
                    self.draw_video_splash()

                 # The event handler loop
                while self.vid.active:      # Only returns True is there is a video currently playing
                    eventHandler.handle_events()

                    if self.opts.enableOSDcurpos:
                        self.opts.enableOSDcurpos = False
                        self.OSD_curPos_flag = not self.OSD_curPos_flag
                        self.draw_OSD_active = (not self.draw_OSD_active if not self.OSD_curPos_flag else True)

                    if self.vid.no_audio:
                        self.mute_flag = True
                        self.key_mute_flag = True

                    if self.mute_flag or self.key_mute_flag or self.vid.no_audio:
                        self.vid.mute()

                    pos_w, pos_h = self.getResolutions()
                    # only draw new frames, and only update the screen if something is drawn
                    # otherwise, if the vid.paused is true (video is paused), then force the display to be
                    # refreshed.
                    if self.vid.draw(self.win, (pos_w, pos_h):
                            force_draw=(False if not self.vid.paused else True)) or self.vid.paused:

                        # other classes of GUI controls udated if necessary
                        ...

                        # whatever controls need to be visible are done in self.update_GUI_components()  (but not all)
                        self.update_GUI_components()
                        pygame.display.update()

                    pygame.time.wait(int(1000 / (self.vid.frame_rate * self.vid.speed)))
                # End of while vid.active
                # Close the object to free up resources.
                self.vid.close()
                if self.forwardsFlag is not True and self.backwardsFlag is not True:
                    # The length of time delay between each video
                    pass
            # End of videoList playback loop
            if self.stopButtonClicked:
                continue
            elif not self.opts.loop:
                break
        # End of the main loop
        self.quit()

Notes on the above:  vid.frame_surf:  The last drawn video frame.  Its a valid frame surface if it is  already drawn if self.vid.draw() returns true. Notice that pygame.display_update() is the last thing done in this if True conditional of self.vid.draw().  Thus the frame is ready to be shown, if any of the gui controls are visible and need updating th is is checked mostly in self.update_GUI_components(). And finally everything is displayed to the display (self.win) by pygame.display_update().
Also note that this application runs in FULLSCREEN mode ONLY.  do the display width and height is the size of the attached monitor.  Smallest monitor supported is 1920x1080, largest is 4K-UHD.

NOTE2: Aplication uses self.vid.change_resolution(self.displayHeight),  This method is defined in the pyvidplayer2 documentation:  Given a height, the video will scale its width while maintaining aspect ratio. Will scale width to an even number.  Currently this function is hardcoded into the application it its use is *required*



ISSUE:  If some GUI controls are showing such as the OSD (current video position in HH:MM:SS format)  or any GUI component that occupies an area of the display that is outside of the video's horizontal or vertical size that is rendered to the display, the contents of that portion of the display in which GUI component occupied will not be erased when told to go invisible. Full erasure of the gui components  work fine if the video is scaled up and occupies the entire screen, or if the video is wide or tall enough to accomidate the GUI control being displayed.  Any GUI control that occupies the actively rendered frame part of the video will refresh with no issue.   Please note that the video is centered onto the display based on its width and height.
